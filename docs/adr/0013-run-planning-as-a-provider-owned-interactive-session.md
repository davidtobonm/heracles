# Run Planning as a provider-owned interactive session

The Planning Stage will launch the selected Planner provider in an inherited interactive terminal for one continuous Grilling Session, preserving the provider's full conversation context while it asks one question at a time. Near the Question Budget, the Planner asks the user whether to exceed it or proceed to PRD creation. The same session remains open while the published PRD Issue awaits approval. The user either tells the provider to approve it, causing the provider to invoke Heracles approval, or approves through another Control Surface and then tells the session to continue. Heracles will not add polling or cross-session approval notifications.

Heracles records the provider session identity when the provider supports resumable conversations, and `heracles resume <labor-id>` reopens that same conversation. Providers without resumable sessions remain usable, but `heracles doctor` warns that an interrupted Grilling Session restarts from persisted problem and PRD Issue context without full conversation history. This replaces the one-shot structured question-and-answer protocol because the quality and continuity of the live interview matters more than a provider-neutral interactive UI.
