# Publish planning output as a PRD Issue

The Planning Stage will use the bundled `to-prd` skill to publish the resolved Grilling Session as a draft PRD Issue in the configured Issue Tracker, matching the established `to-prd` workflow while preserving human control. The PRD Issue keeps one durable URL across revisions: the Planner edits the same issue, labels it `heracles:prd` plus `heracles:review` while it awaits approval, and replaces `heracles:review` with `heracles:approved` when the Planning Approval Gate is approved. Heracles records the URL in durable Labor state, and approval ends the Planner's responsibility and interactive session. Heracles then launches the separately configured Issue Author, which reads the approved issue body before proposing and publishing executable issues in the background without a separate approval gate, as described in ADR-0015.
