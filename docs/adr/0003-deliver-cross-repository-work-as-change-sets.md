# Deliver cross-repository work as Change Sets

Heracles will deliver an issue as a Change Set containing one linked pull request per touched target repository. Pull requests must pass configured local verification and required GitHub CI before Heracles automatically merges them in configured order. Automatic merging is enabled by default, preserving the legacy agent queue's backlog-delivery behavior, and may be explicitly disabled through Project Configuration, Preferences, or a launch override. Because cross-repository delivery cannot be atomic, a merge failure leaves remaining pull requests open and marks the issue blocked.

An issue becomes `heracles:done` only after every Change Set pull request is merged. When automatic merging is explicitly disabled, verified opened pull requests move the issue to `heracles:review`, and the Labor pauses in `awaiting_review`. On resume, Heracles reconciles merged pull requests, marks their issues done, and continues newly unblocked work.

With automatic merging enabled, a required GitHub CI failure or requested pull-request changes trigger an automatic correction cycle against the preserved Issue Workspace. Heracles reruns implementation and review with the failure or review context, pushes corrections, reruns local verification, waits for required CI, and continues the attached Labor. It never merges while required checks or requested changes remain unresolved; exhaustion of the configured retry policy blocks the issue and Labor for operator attention.

The default correction policy permits three cycles per issue and is configurable through `delivery.correction-retries`. Requested changes retry immediately; CI infrastructure failures use short exponential backoff before consuming a correction cycle. Exhaustion blocks the issue and Labor while preserving all work and evidence. Trusted unattended launches may explicitly use `--retry-until-pass`.

Heracles classifies GitHub-reported `cancelled`, `timed_out`, `action_required`, runner-unavailable messages, and verified infrastructure patterns as CI infrastructure failures. Failed test, build, and lint checks are code failures. Unknown failures default to code failures so Heracles attempts correction rather than waiting indefinitely.
